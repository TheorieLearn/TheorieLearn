Constructing G'

To make the newly constructed graph more clear, we will recreate the problem statement with a small example. Start state: $s$, red vertices: R1 and R2, blue vertices: B. {{{params.original_graph}}} From this, we generate our new graph $G' = (V', E')$. Recall from the previous part that $V' = (V \cup \{s\}) \times R \times B \times G$. Now, we need to define the transitions to complete our graph construction.

For simplicity and for this question only, we simplify the problem so that $\mathbf{V' = (V \cup \{s\}) \times R \times B}$ even though the edge definitions for green vertices should be written as well in a real solution. After entering the rules for $E'$ and clicking the submit button, a new $G'$ will be generated based on your given rules. Keep in mind some rules may not affect this example graph but should nonetheless be written to generalize to other graphs.

To write the rules, specify a start vertex to an end vertex. The arrow direction "$\to$" indicates the direction of the edge. Each vertex such as (u, NotSeen, Full) is specified by three components, each coming from sets $V$, $H$, and $C$, just as you defined before.
Please the read the following to learn how to format your inputs. To represent an element from:
  • $V$: Write "u" to represent any outgoing vertex from $G$ and "v" to any matching incoming vertex from $G$. Write $s$ for the sentinel start in $V'$
  • $R$: Write "0" to represent no red vertices encountered and "1" to represent at least one red vertex was encountered. Write "r" to represent a generic element of $R$ (i.e., by using "r", you are saying that the specified edge should be added for both $r = 0$ and $r = 1$).
  • $B$: "0" to represent no blue vertices encountered and "1" to represent at least one blue vertex was encountered. Write "b" to represent a generic element of $R$ (i.e., by using "b", you are saying that the specified edge should be added for both $b = 0$ and $b = 1$).
