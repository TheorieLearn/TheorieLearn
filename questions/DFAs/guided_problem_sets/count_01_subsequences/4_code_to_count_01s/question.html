One important detail to mention is that we can't actually store our counters directly, because these counters are *unbounded*. In order to convert a program into a DFA, the variables we declare in the program need to be restricted to a *finite* number of possible values. (Recall that DFA stands for deterministic *finite* automata.) In this problem, the way to achieve that is by storing the value of the counters *modulo 7* (i.e. the remainder when the counters are divided by 7). Why? Because the language only cares about the value modulo 7.
